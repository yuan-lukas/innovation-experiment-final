# 功能草图说明

## 首页草图说明

首页顶部为统一导航栏，左侧显示平台名称，右侧显示六个页面入口。导航栏下方是大面积蓝色横幅，包含项目主题、简要介绍和两个主要按钮。横幅下方为平台简介与数据统计卡片，再下方为四个主要功能入口卡片。页面底部设置公告区域和统一页脚。首页整体用于展示项目定位和引导用户进入不同模块。

可放入报告的说明文字：  
首页采用“主题展示 + 功能入口 + 公告提示”的布局方式，用户进入网站后能够快速理解平台用途，并通过卡片入口访问课程资源、实验作业、项目展示和学习工具等模块。

## 课程资源页草图说明

课程资源页顶部为页面横幅，主体区域上方设置搜索框，下方使用三列卡片展示课程资源。每张课程卡片包含课程名称、课程简介、学习重点列表和查看详情按钮。卡片下方设置课程学习建议详情区，展示每门课程的推荐练习、项目关联和参考入口。搜索框可根据课程名称或关键词筛选卡片。

可放入报告的说明文字：  
课程资源页通过卡片化方式整理专业课程内容，并加入关键词搜索功能，方便学生快速定位需要复习或参考的课程方向。

## 实验作业页草图说明

实验作业页顶部为页面横幅，主体区域包含搜索框和实验任务列表。实验任务以时间线卡片展示，左侧为实验编号，右侧为完成状态、实验名称、任务说明和技术标签。实验五期末大作业使用重点卡片样式突出。

可放入报告的说明文字：  
实验作业页按照实验进度展示任务内容，突出期末综合项目，能够清晰体现课程学习从基础页面开发到综合项目实现的过程。

## 项目展示页草图说明

项目展示页使用项目卡片网格。每张卡片顶部为项目类型视觉区，中间展示项目名称和项目简介，底部展示技术标签和查看详情按钮。页面下方设置项目详情说明，补充功能构想、技术路线和展示成果。项目类型覆盖静态网站、机器学习、小程序、图像处理、文档管理和项目管理实践。

可放入报告的说明文字：  
项目展示页用于呈现信息学院常见创新项目方向，通过项目简介和技术标签帮助用户理解不同项目的功能特点和技术路线。

## 学习工具页草图说明

学习工具页采用工具卡片布局，每张卡片包含工具简称图标、工具名称、用途说明、适用场景和外部链接按钮。工具覆盖代码编辑、代码托管、技术文档、在线教程、流程图绘制、HTML 校验、Markdown 写作和 Mermaid 流程图预览等学习场景。

可放入报告的说明文字：  
学习工具页将常用开发与学习工具集中展示，为课程项目开发、文档整理和流程图绘制提供统一入口。

## 关于我们页草图说明

关于我们页采用说明卡片布局，包含项目设计目的、选题原因、功能设置依据、完成说明、内容合规说明、隐私与数据来源说明、学习诚信说明。页面风格与其他页面保持一致，适合报告中说明项目背景和规范性。

可放入报告的说明文字：  
关于我们页集中说明项目建设依据和规范要求，体现项目在内容合规、隐私保护、学习诚信和课程实践方面的考虑。
