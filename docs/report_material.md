# 实验报告可用材料

## 实验目的

1. 掌握静态网站项目的需求分析、页面设计、前端实现和运行测试流程。
2. 熟悉 HTML、CSS、JavaScript 在综合页面开发中的应用。
3. 学会使用卡片式布局、统一导航栏、响应式设计和基础交互提升网站完整性。
4. 完成课程期末综合项目的代码整理、文档编写、截图准备和仓库提交。

## 实验设备

- 操作系统：Windows
- 开发工具：Visual Studio Code 或其他文本编辑器
- 浏览器：Microsoft Edge、Google Chrome 或其他现代浏览器
- 主要技术：HTML、CSS、JavaScript、Markdown
- 辅助工具：Git、GitHub、Live Server 插件

## 实验内容

本实验完成“信息学院学生学习导航平台”静态网站项目。网站面向信息学院学生，主要展示课程资源、实验作业、创新项目、学习工具和项目说明。项目共包含首页、课程资源页、实验作业页、项目展示页、学习工具页、关于我们页六个页面。所有页面采用统一导航栏、统一页脚和蓝白色校园风格，并支持电脑端和手机端浏览。

项目实现了导航栏当前页面高亮、课程资源页搜索筛选、实验作业页搜索筛选、首页公告切换、返回顶部按钮和卡片淡入效果。项目不使用后端数据库，不包含登录注册功能，不采集个人隐私数据。

## 项目设计与实现

### 需求分析

信息学院学生在完成课程学习和实验项目时，需要频繁查看课程重点、实验任务、项目案例、开发工具和报告材料。因此项目设计为一个集中式学习导航平台，通过静态网页将分散信息进行分类展示，提高学习资料访问效率。

### 页面结构设计

网站采用多页面结构，根目录下的 `index.html` 为首页，`pages` 文件夹存放课程资源页、实验作业页、项目展示页、学习工具页和关于我们页。`css/style.css` 负责统一视觉样式，`js/main.js` 负责基础交互逻辑，`docs` 文件夹存放项目说明、页面功能说明、业务流程、功能草图、报告材料和截图清单。

### 视觉设计

页面整体采用蓝白色校园风格，蓝色用于导航栏高亮、按钮、页面横幅和重点元素，白色用于卡片背景，浅蓝色用于页面背景和标签。卡片采用统一边框、阴影和圆角，使页面正式、清晰、适合实验报告截图。

### 功能实现

首页展示网站标题、导航栏、横幅、平台简介、主要功能入口、公告区域和页脚。课程资源页展示课程卡片和课程详情说明，并支持关键词搜索。实验作业页展示实验一到实验五的任务列表、任务要点和实验五验收要点，重点突出期末大作业。项目展示页展示多种创新项目卡片，并补充项目详情、功能构想和技术路线。学习工具页展示常用工具链接和适用场景说明。关于我们页说明项目设计目的、选题原因、功能依据、合规说明、隐私与数据来源说明和学习诚信说明。

JavaScript 部分通过 DOM 操作实现当前导航高亮、搜索筛选、公告切换、返回顶部和卡片淡入动画。CSS 部分使用媒体查询实现响应式布局，在手机宽度下导航栏折叠，卡片自动变为单列展示。

## 运行测试

测试方式为直接使用浏览器打开 `index.html`，或使用 VS Code Live Server 插件启动本地服务。测试内容包括：

1. 首页是否能正常显示导航栏、横幅、功能入口、公告区域和页脚。
2. 导航栏是否能正确跳转到各个页面，并高亮当前页面。
3. 课程资源页搜索框输入关键词后，课程卡片是否正确筛选。
4. 实验作业页搜索框输入“实验五”后，是否能突出显示期末大作业相关卡片。
5. 首页公告区域是否能自动切换，圆点按钮是否可点击切换。
6. 页面滚动后返回顶部按钮是否出现，点击后是否返回页面顶部。
7. 页面在电脑端和手机宽度下布局是否正常，无明显文字重叠或内容溢出。

## 总结与体会

通过本次期末综合项目，进一步理解了静态网站从需求分析、页面设计、代码实现到文档整理的完整过程。HTML 负责页面结构，CSS 负责视觉表现和响应式布局，JavaScript 负责基础交互，三者配合能够完成一个较完整的课程项目。项目开发过程中也认识到，规范的目录结构、统一的样式设计、清晰的 README 和完整的截图材料，对课程项目展示和后续维护都非常重要。

## 思政内容

本项目围绕信息学院学生学习场景展开，强调规范学习、合理使用公开资源、保护个人隐私和遵守学术诚信。项目不采集用户隐私数据，不传播不良内容，不使用未经授权的真实用户信息。通过建设学习导航平台，引导学生提高自主学习能力、工程实践能力和规范表达能力，体现专业学习服务成长、技术应用遵守规则、项目开发重视责任的价值导向。
